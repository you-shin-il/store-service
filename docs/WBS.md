## 용어 사전

| 용어                     | 쉬운 설명                                                                         |
|:-------------------------|:----------------------------------------------------------------------------------|
| **엔티티 (Entity)**      | DB 테이블과 1:1 대응되는 Java 클래스. `Product` → `products` 테이블              |
| **Aggregate Root**       | 여러 엔티티를 묶는 대표 엔티티. Order가 대표면 OrderItem은 Order 통해서만 접근    |
| **DTO**                  | 데이터를 실어 나르는 그릇. API 요청/응답용 (엔티티 직접 노출 방지)               |
| **Service**              | 비즈니스 로직을 처리하는 클래스. Controller가 호출하고, Repository를 사용         |
| **Repository**           | DB 접근을 담당하는 인터페이스. Spring Data JPA가 구현체를 자동 생성              |
| **이벤트 (Event)**       | "어떤 일이 일어났다"는 알림. 주문 생성 → `OrderPlacedEvent` → 재고 차감          |
| **프로파일 (Profile)**   | 환경별 설정 전환. `local`=MinIO, `dev`=S3 — 코드 변경 없이 설정만 교체          |
| **JWT**                  | 로그인 후 받는 "출입증". API 호출 시 토큰을 보내면 누구인지 확인 가능            |
| **BFF**                  | Backend For Frontend. 화면별 API 서버 분리 (구매자/판매자/관리자)                |
| **스케줄러 (Scheduler)** | 정해진 시간마다 자동 실행. 예: 매일 자정에 7일 지난 주문 자동 구매확정           |
| **Presigned URL**        | 파일 다운로드용 "임시 링크". 30분 유효, 만료 시 접근 불가                        |
| **소프트 삭제**          | DB에서 실제 삭제 안 하고 상태를 DELETED로 변경. 복구 가능                        |
| **페이지네이션**         | 목록을 "1페이지, 2페이지..."로 나눠서 보여주는 것                                |
